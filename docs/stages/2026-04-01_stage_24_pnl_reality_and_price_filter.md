## 3. Системная проблема: мы входим слишком поздно

### Суть проблемы

Текущая политика разрешает входы в **late/final/stretched** бакеты — то есть когда BTC уже значительно отклонился от открытия. К этому моменту:

1. Рынок уже знает направление → переоценивает YES/NO до 0.92–0.99
2. Наш winrate высокий (94–100%) — потому что это очевидно всем
3. PnL тонкий (+0.8%) — потому что мы покупаем "очевидность"

**Иллюстрация:**

```
late|stretched: BTC ушёл на 0.5%+ от открытия за 3-4 минуты
  → рынок: YES = 0.992 (почти уверен)
  → наш winrate: 100%
  → payout: 1/0.992 = 1.008x
  → PnL: +0.8% ← почти ничего
```

### Ценовая структура по бакетам

| entry_price | payout | нужный winrate для безубытка |
|-------------|--------|------------------------------|
| 0.50–0.60 | 1.67–2.00x | 50–60% |
| 0.60–0.70 | 1.43–1.67x | 60–70% |
| 0.70–0.80 | 1.25–1.43x | 70–80% |
| 0.80–0.90 | 1.11–1.25x | 80–90% |
| **0.90–0.99** | **1.01–1.11x** | **90–99% ← мы здесь** |

## 5. Стратегическое направление

### Текущая архитектура (bucket whitelist)

```
ВХОД если: (time_bucket, distance_bucket) ∈ whitelist И tier ≥ минимальный
```

**Проблема:** вход в late/stretched бакеты = вход тогда, когда рынок уже знает ответ.

### Предложение: ценовой фильтр

```
ВХОД если: signal_tier == very_strong
           AND pm_entry_price < 0.75   ← новое условие
           AND (существующие guards)
```

**Логика:** мы не знаем заранее, в каком бакете будет "справедливая" цена. Вместо hardcode правил — входить только тогда, когда рынок ещё НЕ переоценил направление. Цена 0.75 = рынок оценивает вероятность в 75%, но наш сигнал говорит 80%+ → реальный эдж.

**Ожидаемый результат:**

| сценарий | entry_price | winrate нужный | winrate наш | PnL |
|----------|-------------|----------------|-------------|-----|
| pm_price < 0.60 | 0.55 | 55% | ~65% | ~+18% |
| pm_price < 0.70 | 0.65 | 65% | ~73% | ~+12% |
| pm_price < 0.75 | 0.72 | 72% | ~75% | ~+4% |

### Долгосрочно: `early|near` при истинно нейтральном рынке

Из Level 1A: `early|near` + `very_strong` = 66% winrate. Если рынок при этом оценивает в 0.50–0.55 (истинно нейтральный), PnL = `0.66/0.52 - 1 = +27%`. Нужна сегментация по фактической цене (сейчас mean = 0.649 что говорит о смешанных случаях).

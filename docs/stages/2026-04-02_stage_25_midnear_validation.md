## 4. Валидация гипотезы mid|near

Stage 24 предсказывал для `mid|near` + `very_strong`:

| метрика | прогноз (Stage 24) | факт (Stage 25) |
|---------|-------------------|----------------|
| winrate | 73.4% | 77.2% |
| mean_entry_price | 0.676 | 0.716 |
| expected_pnl | +8.6% | **+7.8%** |

**Гипотеза подтверждена.** Отклонение PnL: −0.8 п.п. — в пределах нормальной вариации.

**Почему mid|near работает:** BTC находится около цены открытия в середине раунда — рынок нейтрален (0.71), но наши детекторы уже фиксируют направление через микроструктуру стакана (order imbalance, wall pull, absorption). Это предиктивный сигнал, а не подтверждающий.

---

## 5. Закономерность: цена входа определяет PnL

Данные двух прогонов подтверждают системную зависимость:

| entry_price диапазон | бакеты | winrate | exp_pnl |
|---------------------|--------|---------|---------|
| < 0.75 | mid\|near, final\|near | 77–86% | **+4–8%** |
| 0.75–0.90 | early\|stretched | 91% | +1–4% |
| > 0.90 | late\|far, mid\|stretched, late\|stretched | 94–100% | **0–1%** |

**Вывод:** высокий winrate при высокой цене = почти нулевой PnL. Реальный эдж — только там, где рынок ещё не переоценил направление.

## 6. Стратегическое направление: ценовой фильтр

### Текущая архитектура (bucket whitelist)

```
ВХОД если: (time_bucket, distance_bucket) ∈ whitelist
           AND signal_tier ≥ минимальный
```

**Проблема:** whitelist разрешает вход в `late|stretched` при цене 0.995 → PnL +0.5%. Это не окупает операционные риски.

### Предложение следующего этапа: ценовой фильтр

```
ВХОД если: signal_tier == very_strong
           AND pm_entry_price < 0.80      ← фильтр по цене
           AND (существующие guards)
```

Это решает проблему системно — без хардкода бакетов. Любой бакет разрешён, если рынок ещё не переоценил исход.

**Ожидаемый эффект:** отсечь `late|stretched` (0.995), `late|far` при высоких ценах, `final|far` при высоких ценах — и сконцентрировать капитал на входах с реальным эджем.

# Project Docs

Этот каталог хранит человекочитаемую документацию по этапам roadmap.

## Правило ведения

- каждый значимый этап работы фиксируется отдельным `.md`;
- stage-документ описывает, что уже реализовано, что является заглушкой и какой следующий шаг;
- аналитические прогоны симулятора сохраняются отдельными markdown-отчётами.

## Структура

- `stages/` — описание завершённых и текущих этапов разработки;
- `reports/` — автоматически или полуавтоматически собранные отчёты по симуляции.
